# Change-set for #25 — Migrate the whole API to async with aiosqlite (every handler, every test)

## Scope

Convert every route handler and DB helper to `async def`, replacing the blocking `sqlite3` stdlib driver with `aiosqlite`. Update `pyproject.toml` to declare the new runtime dep and the new dev dep (`pytest-asyncio`). Rewrite `tests/conftest.py` fixtures that touch the DB layer. All existing (~95) tests must keep passing; no test count change expected.

Out of scope: connection pooling, Postgres migration, asyncio.Lock for rate limiting, any change to JWT/bcrypt code, Trio/anyio.

## Files to touch

- Modify: `pyproject.toml` — add `aiosqlite>=0.19` to `[project].dependencies`; add `pytest-asyncio>=0.23` to `[project.optional-dependencies].dev`; add `asyncio_mode = "auto"` to `[tool.pytest.ini_options]`
- Modify: `src/todo_api/db.py` — replace `get_connection` and `init_schema` with `async def` equivalents; make `get_tags_for_todo` async; keep `_row_to_tag` and `_row_to_todo` as sync pure-mapping helpers
- Modify: `src/todo_api/app.py` — add async `get_db` FastAPI dependency (yields connection, closes in finally); convert all route handlers to `async def`; `await` every SQL call via cursor context manager; make `lifespan` fully async
- Modify: `src/todo_api/auth.py` — convert `get_current_user` to `async def`; `await` the DB user-existence check
- Modify: `src/todo_api/rate_limit.py` — no logic change; `authed_rate_limit` remains sync; FastAPI resolves the async `get_current_user` dep transparently
- Modify: `tests/conftest.py` — switch `isolated_state` fixture to `@pytest_asyncio.fixture(autouse=True) async def`; `await` calls to `db_module.get_connection()` and `db_module.init_schema(conn)`

## Public API / contract changes

HTTP surface is unchanged (same endpoints, same status codes, same JSON shapes). Internal Python API changes:

| Symbol | Before | After |
|---|---|---|
| `db.get_connection` | `def get_connection() -> sqlite3.Connection` | `async def get_connection() -> aiosqlite.Connection` |
| `db.init_schema` | `def init_schema(conn: sqlite3.Connection) -> None` | `async def init_schema(conn: aiosqlite.Connection) -> None` |
| `db.get_tags_for_todo` | `def get_tags_for_todo(conn, todo_id) -> list[Tag]` | `async def get_tags_for_todo(conn, todo_id) -> list[Tag]` |
| `app.get_db` | (does not exist) | `async def get_db() -> AsyncIterator[aiosqlite.Connection]` |
| `auth.get_current_user` | `def get_current_user(...) -> int` | `async def get_current_user(...) -> int` |
| All route handlers in `app.py` | `def` | `async def` |

## Test plan

No new tests. All existing tests keep their `client.get/post/patch/delete(...)` calls unchanged — `TestClient` is sync-facing and handles async routes transparently.

Changes to test infrastructure only:
- `tests/conftest.py`: `isolated_state` becomes an `async def` fixture decorated with `@pytest_asyncio.fixture(autouse=True)`; body awaits `get_connection()` and `init_schema(conn)`.
- `asyncio_mode = "auto"` in `pyproject.toml` so pytest-asyncio detects async fixtures without per-fixture markers.

Acceptance checks from the issue:
- `pytest -q` exits 0, total test count unchanged.
- `grep -cE "^async def" src/todo_api/app.py` > 8.
- `grep -cE "^def \w+.*:" src/todo_api/app.py` is small (only sync helpers like `get_db` generator are excluded; they'll be `async def` too).
- `grep -E "import sqlite3" src/todo_api/*.py` — only `db.py` (for `sqlite3.PARSE_DECLTYPES` and `sqlite3.IntegrityError`).
- `grep -E "import aiosqlite" src/todo_api/*.py` — `db.py` and possibly `app.py`.

## Risk justification

Classified **high**: adds `aiosqlite` as a new runtime dependency (explicit rubric trigger), and is a pervasive refactor touching every handler and every DB call site. Async/sync boundary errors deadlock or panic silently; a single missed `await` breaks runtime correctness without a type error.

## Open questions

None. The issue body is fully specified. The only judgment call is how `_row_to_todo` handles the now-async `get_tags_for_todo`: since `_row_to_todo` itself stays sync, it cannot `await`; the Implementer must inline the `await get_tags_for_todo(conn, row["id"])` call at each call site in `app.py` and pass the resolved list into `_row_to_todo`, or restructure `_row_to_todo` to accept a pre-fetched `tags` argument rather than fetching internally. Either approach is valid; the simplest is to have `_row_to_todo` accept an optional pre-fetched `tags: list[Tag]` and have callers await `get_tags_for_todo` before calling it.
