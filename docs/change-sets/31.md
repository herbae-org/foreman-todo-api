# Change-set for #31 — Migrate from SQLite+aiosqlite to PostgreSQL+asyncpg (with CI service + pool)

## Scope

Replace every SQLite/aiosqlite touchpoint with asyncpg + a connection pool backed by PostgreSQL. Includes schema DDL changes (BIGSERIAL, BOOLEAN, TIMESTAMPTZ, functional unique index), SQL dialect updates across all handlers (`$N` placeholders, `RETURNING`, `fetchrow`/`fetch`/`execute` semantics), CI service container, and per-test Postgres isolation in conftest. No ORM, no Alembic, no new HTTP endpoints, no new business logic.

## Files to touch

- Modify: `pyproject.toml` — replace `aiosqlite>=0.19` with `asyncpg>=0.29` in `[project].dependencies`; add `pytest-asyncio>=0.23` is already present (verify it stays)
- Rewrite: `src/todo_api/db.py` — full replacement: remove aiosqlite/sqlite3, add asyncpg pool (`get_pool`, `close_pool`), new `init_schema(pool)` with Postgres DDL, `get_db` dependency yielding `asyncpg.Connection`, updated `_row_to_todo`/`_row_to_tag`/`get_tags_for_todo`, add `_delete_affected` helper
- Modify: `src/todo_api/app.py` — update all imports (`aiosqlite.Connection` → `asyncpg.Connection`), `lifespan` to use `get_pool`/`close_pool`/`init_schema(pool)`, all SQL strings to use `$N` placeholders, `cursor.lastrowid` → `RETURNING` + `fetchrow`, `cursor.rowcount` → `_delete_affected`/explicit checks, remove `conn.commit()` calls (asyncpg auto-commits by default outside a transaction), update `list_todos` SQL to Postgres-compatible filter logic, `get_stats` `SUM(done)` now works on BOOLEAN column
- Modify: `src/todo_api/auth.py` — replace `get_connection()` import with `get_pool`; rewrite `get_current_user` to `await pool.fetchrow("SELECT id FROM users WHERE id = $1", user_id)`; update login query to `WHERE LOWER(email) = LOWER($1)`; update register insert similarly
- Modify: `tests/conftest.py` — replace `isolated_state` fixture: drop `monkeypatch`/`DB_PATH` approach; use `close_pool`, `get_pool`, `DROP SCHEMA public CASCADE` + `CREATE SCHEMA public` + `init_schema(pool)` pattern; keep `auth_headers` and `bus.reset()` unchanged
- Modify: `.github/workflows/test.yml` — add `services.postgres` block (postgres:16-alpine, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `ports`, health-check options) and `env.DATABASE_URL` at the job level

## Public API / contract changes

No HTTP endpoint changes. Internal DB interface changes:

| Before | After |
|---|---|
| `db.get_connection() -> aiosqlite.Connection` | removed; replaced by `db.get_db()` FastAPI dependency |
| `db.init_schema(conn: aiosqlite.Connection)` | `db.init_schema(pool: asyncpg.Pool)` |
| `db.DB_PATH: Path` | removed; replaced by `db.DATABASE_URL: str` |
| `db.IntegrityError` (re-export of `sqlite3.IntegrityError`) | removed; catch `asyncpg.UniqueViolationError` directly in handlers |
| `conn.execute(...).lastrowid` pattern | `conn.fetchrow("... RETURNING id, ...")` |
| `cursor.rowcount` pattern | `int((await conn.execute(...)).split()[-1])` via `_delete_affected` |
| `bool(row["done"])` in `_row_to_todo` | `row["done"]` directly (asyncpg returns native Python `bool`) |
| `datetime.fromisoformat(row["created_at"])` | `row["created_at"]` directly (asyncpg returns `datetime` with tzinfo) |

## Risk justification

Classified **high**: this change swaps the runtime storage engine (new `asyncpg>=0.29` dependency), rewrites the database schema DDL, modifies the CI pipeline by adding a service container, and touches every SQL query across `app.py` and `auth.py`. Three of the five high-risk triggers apply simultaneously (new runtime dependency, database schema change, CI infrastructure change).

## Open questions

1. **`list_todos` done filter:** The SQLite version passes `done_val = int(done)` and uses `? IS NULL OR t.done = ?`. In Postgres, `done` is BOOLEAN and `$N IS NULL` is valid, but passing Python `None` vs `True`/`False` through asyncpg needs verification — implementer should test this filter path explicitly.
2. **`get_stats` SUM(done):** SQLite stored `done` as `INTEGER` so `SUM(done)` worked. In Postgres `SUM(done::int)` or `COUNT(*) FILTER (WHERE done)` is the correct idiom — implementer should choose and update accordingly.
3. **`assign_tags` INSERT OR IGNORE:** SQLite-specific; Postgres equivalent is `INSERT INTO todo_tags ... ON CONFLICT DO NOTHING`. Implementer must update this.
4. **`list_todos` `has_tag_filter` trick:** The `AND (? = 0 OR tt.tag_id IN (...))` pattern uses an integer flag. In Postgres this should still work but `$N` numbering gets complex with variadic `tag_ids`. Implementer should trace the parameter positions carefully.
5. **`pytest-asyncio` + asyncpg fixture scope:** The `isolated_state` fixture needs `@pytest_asyncio.fixture(autouse=True)` and an event loop — confirm `asyncio_mode = "auto"` in `pyproject.toml` covers this without additional scope changes.
6. **`asyncpg.UniqueViolationError` import:** The issue says remove `IntegrityError` re-export. Implementer should catch `asyncpg.UniqueViolationError` in the `register` and `create_tag` handlers directly rather than going through a re-export.
