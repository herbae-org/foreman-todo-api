# Change-set for #34 — Replace raw asyncpg SQL with SQLAlchemy 2.0 async ORM + Alembic migrations

## Scope

Replace every raw `conn.execute` / `conn.fetch` / `pool.fetchrow` call across `db.py`, `app.py`, and `auth.py` with SQLAlchemy 2.0 async ORM queries, introduce `models.py` with declarative table models, and wire Alembic for a single baseline migration that replaces `init_schema()`. `events.py`, `rate_limit.py`, and the WebSocket handler are untouched. The public HTTP API contract is unchanged.

## Files to touch

- Create: `src/todo_api/models.py` — SQLAlchemy `DeclarativeBase` + `User`, `Todo`, `Tag`, `TodoTag` mapped classes
- Modify: `src/todo_api/db.py` — replace asyncpg pool helpers with `AsyncEngine`/`async_sessionmaker` + `get_db` DI provider; drop `init_schema`, `get_tags_for_todo`, `_row_to_tag`, `_row_to_todo`, `_delete_affected`, `terminate_all_pools`
- Modify: `src/todo_api/app.py` — replace all `asyncpg.Connection` dependencies and raw SQL with ORM session calls; drop imports of removed db helpers
- Modify: `src/todo_api/auth.py` — replace `get_pool().fetchrow` with `AsyncSession` + ORM `select(models.User.id)` query
- Create: `alembic.ini` — standard alembic config; `sqlalchemy.url` set as a placeholder (overridden by `env.py` at runtime)
- Create: `alembic/env.py` — async alembic environment that reads `DATABASE_URL` from `os.environ`, imports `Base` from `todo_api.models`, and targets `Base.metadata`
- Create: `alembic/versions/0001_baseline.py` — autogenerated baseline migration matching the current schema (users, todos, tags, todo_tags with correct indexes and constraints)
- Modify: `pyproject.toml` — add `sqlalchemy[asyncio]>=2.0,<3.0` and `alembic>=1.13` to `[project].dependencies`; retain `asyncpg>=0.29` (still the async driver under SQLAlchemy)
- Modify: `tests/conftest.py` — add session-scoped `_run_migrations_once` fixture (runs `alembic upgrade head` once per pytest session); replace the per-test `isolated_state` body: drop `terminate_all_pools`/`init_schema` calls, use an `AsyncSession` to `TRUNCATE users, todos, tags, todo_tags RESTART IDENTITY CASCADE`; drop direct `asyncpg.connect` import
- Modify: `.github/workflows/test.yml` — insert `alembic upgrade head` step (with `DATABASE_URL` env) between `pip install -e .[dev]` and `pytest -q`

## Public API / contract changes

None. All HTTP endpoints (`/health`, `/auth/*`, `/todos`, `/todos/{id}`, `/tags`, `/todos/{id}/tags`, `/ws/todos`) retain identical paths, methods, status codes, and JSON shapes.

Internal Python API changes (not part of the public HTTP contract):

| Before | After |
|---|---|
| `db.get_pool() -> asyncpg.Pool` | removed |
| `db.close_pool() -> None` | `db.close_engine() -> None` |
| `db.terminate_all_pools() -> None` | removed |
| `db.init_schema(conn) -> None` | removed (Alembic handles DDL) |
| `db.get_db() -> AsyncIterator[asyncpg.Connection]` | `db.get_db() -> AsyncIterator[AsyncSession]` |
| `db.get_tags_for_todo(conn, todo_id)` | removed (tags loaded via `lazy="selectin"` on `Todo.tags`) |
| `db._row_to_tag(row)` | removed |
| `db._row_to_todo(row, tags)` | removed |
| `db._delete_affected(conn, sql, *args)` | removed |
| `auth.get_current_user` uses `get_pool()` directly | uses `AsyncSession = Depends(get_db)` |

## Test plan

**Existing tests (~105) must all pass unchanged at the assertion level.** Structural changes required:

- `conftest.py` must add a session-scoped `_run_migrations_once` fixture (`autouse=True`, `scope="session"`) that calls `alembic upgrade head` before any test runs.
- The per-test `isolated_state` autouse fixture replaces its body: instead of `terminate_all_pools` + raw `asyncpg.connect` + `init_schema`, it acquires an `AsyncSession` from `await db_module.get_sessionmaker()` and executes `TRUNCATE users, todos, tags, todo_tags RESTART IDENTITY CASCADE` + `await session.commit()`, then calls `rate_limit.reset_buckets()` and `bus.reset()`.
- Any test importing `db.get_connection`, `db.close_pool`, `db.terminate_all_pools`, or `asyncpg.UniqueViolationError` directly must be updated to the new equivalents (`get_db`, `close_engine`, `sqlalchemy.exc.IntegrityError`). Most such assertions are already tested at the HTTP level (409/422 responses), so very few changes expected.
- No new test files needed; no new test cases required. The migration fixture and isolation fixture carry all structural change.

## Risk justification

Rated **high**: introduces two new runtime dependencies (`sqlalchemy[asyncio]`, `alembic`), rewrites the entire database access layer, touches authentication, and adds a CI migration step — all of which appear in the high-risk rubric (new runtime deps, database migrations, auth, CI pipeline change).

## Open questions

1. **`alembic/versions/0001_baseline.py` source of truth**: The issue says to generate the baseline via `alembic revision --autogenerate`. The Implementer should run this against a live Postgres instance during development; if the generated migration diverges from the raw DDL in the current `init_schema()` (e.g., index names), the generated file wins and any mismatch should be noted in the PR description.
2. **`LOWER(name)` unique index vs plain `UNIQUE(user_id, name)`**: The issue explicitly replaces the functional index with a plain unique constraint and shifts the `lower()` call to handler code. The baseline migration should reflect this new constraint shape, not reproduce the old functional index — confirm this is intentional before merging (it is a silent behavioural change: duplicate tag names differing only in case that somehow exist in production would become a migration blocker).
3. **`alembic.ini` `sqlalchemy.url` value**: Standard practice is to set a dummy placeholder in `alembic.ini` and override in `env.py` from `os.environ["DATABASE_URL"]`. Confirm the Implementer should not rely on the `.ini` value for anything — `env.py` is authoritative.
4. **`asyncpg` explicit dep retention**: The issue says "your call — be consistent." Given that SQLAlchemy's asyncio extras install asyncpg transitively, keeping it explicit in `pyproject.toml` avoids a surprise pin change if SQLAlchemy ever changes its default async driver. Recommend keeping it explicit.
5. **`auth.py` session dependency**: `get_current_user` currently acquires a pool connection directly (`get_pool().fetchrow`). After this change it must accept `session: AsyncSession = Depends(get_db)` as a parameter. This is a parameter-level change to an existing FastAPI dependency; confirm that the rate-limit chain (`authed_rate_limit` → `get_current_user`) still composes correctly with the added session dependency, since FastAPI DI resolves each dependency independently and the session may be acquired twice per request (once in `get_current_user`, once in the handler). This is fine for correctness but worth flagging for efficiency.
