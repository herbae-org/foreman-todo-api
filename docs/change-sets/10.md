# Change-set for #10 — Complete the CRUD: PATCH, DELETE, done filter, and stats endpoint

## Scope

Add four new capabilities to `src/todo_api/app.py`: `PATCH /todos/{id}` (partial update), `DELETE /todos/{id}`, a `done` filter query parameter on `GET /todos`, and a `GET /todos/stats` aggregate endpoint. All four ship with dedicated test files. SQLite persistence, id re-use after delete, and bulk endpoints are explicitly out of scope.

## Files to touch

- Modify: `src/todo_api/app.py` — add `PatchTodo` Pydantic model, extend `list_todos` with `done` filter param, insert `GET /todos/stats` route before `GET /todos/{id}`, add `PATCH /todos/{id}` and `DELETE /todos/{id}` handlers.
- Create: `tests/test_todos_patch.py` — 8 tests covering PATCH happy paths, 404, 422 (empty title, extra fields), and `created_at` immutability.
- Create: `tests/test_todos_delete.py` — 4 tests covering 204 empty body, subsequent 404, missing-id 404, and total reduction.
- Create: `tests/test_todos_filter.py` — 6 tests covering `done=true/false`, omitted param, ordering, pagination interaction, and invalid value 422.
- Create: `tests/test_todos_stats.py` — 4 tests covering empty store, all-pending, mixed, and parametrised `total == done + pending` invariant.

## Public API / contract changes

New endpoints added (none removed, none changed in a breaking way):

| Before | After |
|---|---|
| `GET /health` | `GET /health` (unchanged) |
| `POST /todos` | `POST /todos` (unchanged) |
| `GET /todos?limit&offset` | `GET /todos?limit&offset&done` — `done` is optional; `total` now reflects filtered count when `done` is provided |
| `GET /todos/{id}` | `GET /todos/{id}` (unchanged) |
| — | `GET /todos/stats` → `{"total":int,"done":int,"pending":int}` |
| — | `PATCH /todos/{id}` body `{"title?":str,"done?":bool}` → `Todo` 200 / 404 / 422 |
| — | `DELETE /todos/{id}` → 204 empty / 404 |

New Pydantic model `PatchTodo`:
```python
class PatchTodo(BaseModel):
    model_config = {"extra": "forbid"}
    title: str | None = Field(default=None, min_length=1, max_length=200)
    done: bool | None = None
```

`list_todos` signature change:
```python
# Before
def list_todos(limit: int = Query(...), offset: int = Query(...)) -> TodoList

# After
def list_todos(limit: int = Query(...), offset: int = Query(...), done: bool | None = Query(default=None)) -> TodoList
```

Note on `total` semantics change: previously `total` was always `len(_todos)`; after this change, when `done` is supplied `total` reflects the filtered count. Existing tests that omit `done` are unaffected because `done=None` returns all items and `total` equals `len(_todos)`.

Note on route order: `GET /todos/stats` must be registered **before** `GET /todos/{id}` so FastAPI matches the literal path first. In `app.py`, insert the stats handler above the existing `get_todo` handler.

## Test plan

New test files (40 total passing tests expected):

**`tests/test_todos_patch.py`** (8 tests):
1. `test_patch_updates_title` — POST, PATCH `{"title":"new"}`, assert title changed, done unchanged.
2. `test_patch_marks_done` — POST, PATCH `{"done":true}`, assert done=true, title unchanged.
3. `test_patch_both_fields_at_once` — PATCH `{"title":"x","done":true}`, both applied.
4. `test_patch_empty_body_is_noop` — PATCH `{}`, 200, item unchanged.
5. `test_patch_missing_id_returns_404` — PATCH `/todos/999` on empty store → 404 `{"detail":"Not Found"}`.
6. `test_patch_rejects_empty_title` — PATCH `{"title":""}` → 422.
7. `test_patch_rejects_extra_fields` — PATCH `{"created_at":"2020-01-01T00:00:00Z"}` → 422.
8. `test_patch_preserves_created_at` — POST, PATCH title, assert `created_at` identical.

**`tests/test_todos_delete.py`** (4 tests):
1. `test_delete_removes_item` — POST, DELETE, assert 204; subsequent GET `/todos/1` → 404.
2. `test_delete_returns_204_empty_body` — assert `status_code==204` AND `content==b""`.
3. `test_delete_missing_returns_404` — DELETE on empty store → 404.
4. `test_delete_reduces_total` — POST 3, DELETE 1, `GET /todos` → `total==2`.

**`tests/test_todos_filter.py`** (6 tests):
1. `test_filter_done_true` — POST 3, mark 2 done, `?done=true` → 2 items, `total=2`.
2. `test_filter_done_false` — same setup, `?done=false` → 1 item, `total=1`.
3. `test_filter_omitted_returns_all` — same setup, no param → 3 items, `total=3`.
4. `test_filter_preserves_order` — `?done=true` returns items in id DESC order.
5. `test_filter_interacts_with_pagination` — POST 5, mark 3 done, `?done=true&limit=2` → 2 items, `total=3`.
6. `test_filter_invalid_value_returns_422` — `?done=maybe` → 422.

**`tests/test_todos_stats.py`** (4 tests):
1. `test_stats_empty_store` — `GET /todos/stats` → `{"total":0,"done":0,"pending":0}`, 200.
2. `test_stats_after_posts` — POST 5, all pending → `total=5, done=0, pending=5`.
3. `test_stats_after_mixed` — POST 5, mark 2 done → `total=5, done=2, pending=3`.
4. `test_stats_invariant` — parametrised `(total,done_count)` in `(0,0),(1,0),(1,1),(5,2),(10,10)`: assert `stats.total == stats.done + stats.pending`.

Existing 18 tests (health + create + read) must pass unchanged. The `test_list_returns_total_regardless_of_pagination` test omits the `done` param so `total` still equals `len(_todos)` — no conflict.

Each new test file must use the same `_reset_todos` monkeypatch fixture pattern as existing test files (`monkeypatch.setattr(app_module, "_todos", [])`).

## Risk justification

This change adds four new HTTP endpoints and modifies the signature of an existing one (`GET /todos`), which constitutes a public API surface change. Per the risk rubric, any public API surface change — breaking or additive — is classified as high.

## Open questions

None. The issue body is comprehensive and covers all edge cases including the FastAPI routing gotcha for `/todos/stats` vs `/todos/{id}`, the `total` semantics change for filtered responses, `created_at` immutability via `extra="forbid"`, and the deliberate deferral of id re-use after deletion.
