# Change-set for #13 — Migrate storage from in-memory list to SQLite (preserve all existing behavior)

## Scope

Replace the module-level `_todos` list in `app.py` with SQLite-backed persistence using the stdlib `sqlite3` module. All existing HTTP endpoint contracts, Pydantic models, and test assertions remain unchanged. Out of scope: async SQLite, migrations tooling, connection pooling, any schema beyond the `todos` table, new runtime dependencies.

## Files to touch

- Create: `src/todo_api/db.py` — new module encapsulating SQLite connection, schema init, and row-to-model conversion
- Modify: `src/todo_api/app.py` — remove `_todos`/`_next_id`, add startup hook, rewrite all six handlers to use parameterized SQL via `db.py`
- Modify: `tests/test_todos_create.py` — replace `_reset_todos` fixture with `isolated_db` that monkeypatches `DB_PATH` to a tmp file
- Modify: `tests/test_todos_read.py` — same fixture swap
- Modify: `tests/test_todos_patch.py` — same fixture swap
- Modify: `tests/test_todos_delete.py` — same fixture swap
- Modify: `tests/test_todos_stats.py` — same fixture swap
- Modify: `tests/test_todos_filter.py` — same fixture swap

`tests/test_health.py` requires no changes (no store-reset fixture).

## Public API / contract changes

None. All six endpoints (`GET /health`, `POST /todos`, `GET /todos`, `GET /todos/{id}`, `PATCH /todos/{id}`, `DELETE /todos/{id}`) keep their existing HTTP method, path, query parameters, request body shape, response body shape, and status codes.

## Test plan

No new tests. All existing tests must continue to pass. The only change to test files is the fixture:

```python
# Before (in each test file):
from todo_api import app as app_module

@pytest.fixture(autouse=True)
def _reset_todos(monkeypatch: pytest.MonkeyPatch) -> None:
    monkeypatch.setattr(app_module, "_todos", [])

# After (in each test file):
from todo_api import db as db_module

@pytest.fixture(autouse=True)
def isolated_db(tmp_path: pytest.TempPathFactory, monkeypatch: pytest.MonkeyPatch) -> None:
    monkeypatch.setattr(db_module, "DB_PATH", tmp_path / "test.db")
    db_module.init_schema(db_module.get_connection())
```

Do NOT introduce a shared `conftest.py` — keep the fixture duplicated across the six affected test files as directed.

### Key SQL correctness points the implementer must verify

- `GET /todos` done-filter: use `(? IS NULL OR done = ?)` with the done value passed twice; pass `None`/`None` when omitted.
- `GET /todos` total: use a separate `SELECT COUNT(*) FROM todos WHERE (? IS NULL OR done = ?)` with the same filter params, not `len(rows)` after slicing.
- `PATCH /todos/{id}` no-op: `COALESCE(?, title)` with `None` correctly preserves existing value; `rowcount == 1` still holds because the UPDATE touches the row.
- `PATCH /todos/{id}` done=False: `COALESCE(0, done)` returns `0` (not NULL), so explicitly patching to false works correctly.
- `DELETE` / `PATCH` 404: gate on `cursor.rowcount == 0` after the statement executes.
- `POST` new id: read from `cursor.lastrowid`, not a counter.
- `created_at` round-trip: store as ISO 8601 string (`datetime.isoformat()`); re-hydrate with `datetime.fromisoformat()` in `_row_to_todo`.
