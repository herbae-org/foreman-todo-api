# Change-set for #19 ‚Äî Multi-user auth: JWT bearer tokens with per-user data isolation

## Scope

Convert the single-tenant TODO API to a multi-tenant one by adding JWT-based authentication, a `users` table, and scoping every existing endpoint to the authenticated user. New endpoints: `POST /auth/register`, `POST /auth/login`, `GET /auth/me`. Out of scope: refresh tokens, OAuth, email verification, password reset, admin role, `DELETE /auth/me`, rate limiting, any async code.

## Public API / contract changes

**New endpoints:**

| Method | Path | Auth required | Response |
|--------|------|---------------|----------|
| `POST` | `/auth/register` | No | 201 `{"id":N,"email":"...","created_at":"..."}` / 409 on duplicate |
| `POST` | `/auth/login` | No | 200 `{"access_token":"...","token_type":"bearer","expires_in":3600}` / 401 on bad creds |
| `GET`  | `/auth/me` | Yes | 200 `{"id":N,"email":"...","created_at":"..."}` / 401 without token |

**Breaking changes to existing endpoints:**

All endpoints except `GET /health`, `POST /auth/register`, `POST /auth/login` now require `Authorization: Bearer <token>`. Requests without a valid token ‚Üí 401. Requests for another user's resource ‚Üí 404 (not 403).

`POST /tags` duplicate-email constraint changes from global uniqueness to per-user uniqueness (`UNIQUE(user_id, name)`).

**Schema change (breaking):**

```sql
-- Before
CREATE TABLE todos (id, title, done, created_at)
CREATE TABLE tags  (id, name UNIQUE COLLATE NOCASE)

-- After
CREATE TABLE users (id, email UNIQUE COLLATE NOCASE, password_hash, created_at)
CREATE TABLE todos (id, user_id NOT NULL REFERENCES users(id) ON DELETE CASCADE, title, done, created_at)
CREATE TABLE tags  (id, user_id NOT NULL REFERENCES users(id) ON DELETE CASCADE, name, UNIQUE(user_id, name))
CREATE INDEX idx_todos_user ON todos(user_id)
CREATE INDEX idx_tags_user  ON tags(user_id)
```

**New module `src/todo_api/auth.py` ‚Äî public surface:**

```python
JWT_SECRET: str                          # from env or test fallback
JWT_ALGORITHM: str = "HS256"
JWT_EXPIRY_SECONDS: int = 3600

def hash_password(plain: str) -> str
def verify_password(plain: str, hashed: str) -> bool
def create_token(user_id: int) -> str
def decode_token(token: str) -> int      # raises jwt.InvalidTokenError subtypes
def get_current_user(authorization: str = Header(...)) -> int  # FastAPI dependency
```

## Risk justification

Classified **high**: touches authentication/authorization (new JWT + bcrypt stack), breaking database schema change (`user_id` FK added to `todos` and `tags`), new runtime dependencies (`passlib[bcrypt]`, `pyjwt`, `pydantic[email]`), and all existing public API endpoints now require a bearer token ‚Äî a breaking API surface change.

## Open questions

1. The `isolated_db` fixture currently uses `monkeypatch.setattr(db_module, "DB_PATH", ...)` per file. After moving to `conftest.py`, the same technique applies at the session scope or function scope ‚Äî implementer should confirm function scope (one fresh DB per test) is sufficient given the isolation tests need two users in the same DB.

2. The issue says the `get_current_user` dependency should "verify the user still exists via a db lookup." This creates a DB round-trip on every authenticated request. No caching mechanism is in scope ‚Äî confirm implementer accepts this overhead.

3. `test_users_cannot_attach_others_tags` ‚Äî the issue says "400 `Unknown tag_id: 1` OR 404 on todo (depending on check order ‚Äî either is acceptable, document which)." The change-set proposes documenting the current code order (todo existence first, then tag validation) ‚Üí user B sees 404 on the todo since user B does not own user A's todo. Implementer should verify and set the test assertion accordingly.

4. The existing `test_create_tag_duplicate_case_insensitive_returns_409` test registers two tags with the same case-insensitive name for the same user and expects 409. After the schema change to `UNIQUE(user_id, name)` with `COLLATE NOCASE`, this behavior is preserved per user ‚Äî confirm the SQLite UNIQUE clause supports COLLATE NOCASE on composite unique constraints or needs the constraint defined differently.
